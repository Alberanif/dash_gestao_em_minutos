# Instagram Data Collection Fix - Investigation Summary

## Problem Resolved ✓

**Root Cause:** Data table mismatch
- Collection code writes to: `dash_gestao_instagram_profile_daily` (new table)
- Dashboard was reading from: `dash_gestao_instagram_profile_snapshots` (old table)
- Result: Collected data was invisible to dashboard

## Solution Implemented

### API Endpoint Migration
1. **`/api/instagram/profile`** now reads from `profile_daily`
   - Auto-detects first active Instagram account if account_id not provided
   - Maps `date` field to `collected_at` for backward compatibility
   - Queries last 30 days by default

2. **`/api/instagram/media`** now reads from `media_daily`
   - Same account auto-detection behavior
   - Orders by `last_collected_at DESC` to show newest media first
   - Deduplicates by media_id

### Code Changes
- `src/app/api/instagram/profile/route.ts` - migrated to profile_daily
- `src/app/api/instagram/media/route.ts` - migrated to media_daily

## Expected Behavior After Fix

✓ **Followers data:** Should now display correctly (was showing due to snapshot fallback)
✓ **Followers graph:** Should plot historical followers over time
✓ **Reach data:** Will display collected reach metrics (if available from Instagram API)

### Issue: Reach showing as zero
**Possible causes:**
1. Instagram insights API has 24-48h delay before data becomes available
2. Insights endpoint may not return data for the current day
3. API may require specific permissions/account type

### Issue: Impressions always zero
**Root cause:** Intentional - Instagram's profile insights endpoint doesn't provide impressions metric
- Commit 28e7eb4 removed impressions from API calls (by design)
- Only `reach` is available from profile-level insights
- Media-level data may have different metrics available

**Status:** This is working as designed. Instagram API limitation.
